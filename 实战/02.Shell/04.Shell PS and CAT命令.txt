PS命令总结

不错的一个网站：http://linux.51yip.com/search/ps  命令的详细使用

Linux中的ps命令是Process Status的缩写。ps命令用来列出系统中当前运行的那些进程。ps命令列出的是当前那些进程的快照，
就是执行ps命令的那个时刻的那些进程，如果想要动态的显示进程信息，就可以使用top命令。

一般使用方式
BSD 格式 —— 参数前面不加 -
ps aux

UNIX/LINUX 格式 —— 参数前面通常要加 -
ps -ef

ps aux 信息更详细
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
USER

设置显示哪些
# ps -e -o pid,stat,pcpu

BSD模式
# ps ax o pid,stat


那个用户运行的进程
PID
进程的pid

%CPU
进程占用cpu百分比

%MEM
进程占用内存百分比

VSZ
该进程使用掉的虚拟内存量（KB）

RSS
该进程所占用掉的固定内存量

TTY
在哪个终端运行的进程tty1，tty2。？代表后台运行。pts/0代表终端

STAT
进程的状态。S:休眠的。s:主进程。<:优先级高。N:优先级低。R:运行的。+:前台的。T:暂停的。D:不可中断的。L:被内存锁住的。l：多线程进程。X:死掉的进程。Z:僵尸进程。

START
该进程被触发启动时间

TIME
占用cpu时间

COMMAND
进程名、命令名。


ps命令基本选项及参数释义
a：显示现行终端机下的所有程序，包括其他用户的程序。
c：列出程序时，显示每个程序真正的指令名称，而不包含路径，选项或常驻服务的标示。
e：列出程序时，显示每个程序所使用的环境变量。
f：用ASCII字符显示树状结构，表达程序间的相互关系。
g：显示现行终端机下的所有程序，包括群组领导者的程序。
h：不显示标题列。
u：以用户为主的格式来显示程序状况。
x：显示所有程序，不以终端机来区分。
r：只列出现行终端机正在执行中的程序。
v：采用虚拟内存的格式显示程序状况
-a：显示所有终端机下执行的程序，除了阶段作业领导者之外。
-c：显示CLS和PRI栏位。
-d：显示所有程序，但不包括阶段作业领导者的程序。
-e：显示所有程序。
-f：显示UID,PPIP,C与STIME栏位。
-H：显示树状结构，表示程序间的相互关系。
-u<用户识别码>：列出属于该用户的程序的状况，也可使用用户名称来指定。
-j：采用工作控制的格式显示程序状况
-l或l：采用详细的格式来显示程序状况。
-N：显示所有的程序，除了执行ps指令终端机下的程序之外。

CAT 命令
1.输出内容到文件 cat '' > file
2.输出内容 cat file
3.合并文件 cat f1 f2 > f3
4.合并多个压缩包 cat cat test.tar.gz_?? > test.tar.gz
5.编辑文件
cat >> f1 <<END
ASSS
DASAD
END
6.编辑文件
cat > aa

题目 查看文件第20行到第三十行
解决方式1： cat file|head -30|tail -10
解决方式2： sed -n 20,30p file
